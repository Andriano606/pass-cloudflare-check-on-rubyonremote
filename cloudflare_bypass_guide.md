# Документація: Обхід Cloudflare та Anti-Bot систем (Повна версія)

Цей документ описує багаторівневий підхід до автоматизації браузера, який дозволяє імітувати реального користувача та уникати детекції.

## 1. Рівень Інфраструктури (IP та Мережа)

Cloudflare аналізує вашу мережеву активність ще до завантаження JS.

*   **IP Reputation**: Адреси дата-центрів мають низьку репутацію. Найкращий варіант — **резидентні або мобільні проксі**.
*   **TLS Fingerprinting (JA3/JA4)**: Стек протоколу TLS має бути ідентичним справжньому браузеру. Використання `channel: 'chrome'` у Playwright автоматично використовує мережевий стек встановленого Chrome.
*   **WebRTC Leak**: Вимкніть WebRTC або використовуйте проксі, що підтримує UDP, щоб Cloudflare не зміг побачити вашу реальну внутрішню IP-адресу через WebRTC запити.

## 2. Рівень Браузера (Конфігурація)

### Використання справжнього браузера
Завжди використовуйте `channel: 'chrome'` або `'msedge'`. Стандартний Chromium без кодеків та специфічних Google-компонентів детекується дуже легко.

### Стійкий контекст (Persistent Context)
Використання `launch_persistent_context` створює реальний профіль з кешем та куками. Це дозволяє проходити перевірки на основі "довіри до старого клієнта".

### Прапорці запуску (Args)
*   `--disable-blink-features=AutomationControlled`: Головний прапорець для приховування `navigator.webdriver`.
*   `--disable-features=IsolateOrigins,site-per-process`: Іноді допомагає при роботі з фреймами Cloudstile (Turnstile).

## 3. Рівень JavaScript (Stealth Injection)

### Просунуті перевірки об'єктів:
1.  **`navigator.webdriver`**: Має бути повністю видалений або встановлений у `undefined`.
2.  **Window Dimensions**: У справжнього браузера `window.outerWidth > window.innerWidth` (через рамки вікна). Якщо вони рівні — це ознака headless-режиму.
3.  **Hardware Concurrency**: `navigator.hardwareConcurrency` має відповідати реалістичним значенням (4, 8, 16). Значення 1 або 2 на потужних сайтах виглядають підозріло.
4.  **Device Memory**: `navigator.deviceMemory` має бути узгодженим з типом пристрою в User-Agent.
5.  **WebGL Vendor/Renderer**: Має бути "Google Inc. (Intel)" або "NVIDIA...", а не "Mesa" чи "SwiftShader".

## 4. Узгодженість даних (Consistency)

Це найважливіший пункт, на якому «паляться» більшість скриптів.

*   **User-Agent vs Client Hints**: Якщо ви змінили UA, ви ПОВИННІ змінити `Sec-CH-UA` заголовки.
*   **Timezone vs IP**: Часовий пояс у браузері (`Intl.DateTimeFormat().resolvedOptions().timeZone`) має збігатися з геолокацією вашої IP-адреси.
*   **Languages**: `navigator.languages` має відповідати країні вашого проксі.

## 5. Поведінковий аналіз та Тімінг

Cloudflare Turnstile аналізує динаміку.

*   **Mouse Movements**: Використовуйте криві Безьє або бібліотеки типу `ghost-cursor` для імітації нелінійного руху миші.
*   **Keyboard**: Не вставляйте текст миттєво через `element.fill`. Використовуйте `page.keyboard.type` з випадковими затримками між натисканнями (50-150 мс).
*   **Idle Time**: Не закривайте браузер миттєво після отримання даних. Дайте сторінці «подихати» 1-2 секунди, імітуючи читання контенту.

## Чек-лист для перевірки:

1.  [ ] Використовується `channel: 'chrome'` (справжній бінарний файл).
2.  [ ] Активовано `--disable-blink-features=AutomationControlled`.
3.  [ ] Налаштовано `Persistent Context` з папкою профілю.
4.  [ ] Часовий пояс браузера збігається з геолокацією IP.
5.  [ ] Додано stealth-скрипт для плагінів, WebGL та `navigator.webdriver`.
6.  [ ] У заголовках додані Client Hints (`Sec-CH-UA`).
7.  [ ] Імітується рух миші та скролінг.
8.  [ ] Роздільна здатність екрана та вікна імітує реальний монітор (напр. 1920x1080).
