# Посібник з автоматизації форм (Vue.js / Vuetify) за допомогою Playwright

Цей посібник створено на основі успішного досвіду автоматизації подачі заявок на вакансії на платформі Hurma (Vue.js + Vuetify), яка захищена Cloudflare та Google reCAPTCHA v3.

Сучасні SPA-додатки (Single Page Applications) кардинально відрізняються від класичних сайтів, тому стандартні підходи часто дають збій.

---

## 1. Пошук форм на сторінці

У Vue-додатках форми часто рендеряться динамічно, а класичний тег `<form>` може взагалі бути відсутнім (сабміт відбувається через XHR/Fetch).

**Алгоритм дій:**
1. **Перевірка наявності:** Спочатку спробуйте знайти стандартний тег `<form>` або контейнери типу `.form-container`, `.response-form`.
2. **Фолбек на інпути:** Якщо явного тегу форми немає, шукайте обгортку, яка містить `input` або `textarea`.
3. **Обробка динамічного завантаження:** Якщо форми немає на сторінці, можливо, це сторінка опису. Шукайте кнопки для виклику форми: `page.locator('button:has-text("Apply"), a:has-text("Відгукнутися")')`.
4. **Очікування:** Після кліку обов'язково давайте затримку (`sleep` або `wait_for_selector`), оскільки Vue може плавно відрендерити форму з анімацією або підтягнути її по API.

---

## 2. Заповнення полів (Специфіка Vue / Vuetify)

Фреймворки на кшталт Vuetify використовують кастомні компоненти, які "ламають" стандартні методи Playwright. 

### Прості інпути (Text, Email, URL, Numbers)
Vue-компоненти часто використовують жорстку валідацію на стороні клієнта (Vuelidate).
- **Як шукати:** Перебирайте `input` і зчитуйте їх `type` або `placeholder` (напр., `.get_attribute('placeholder')`).
- **Заповнення:** Використовуйте `element.fill(value)`.
- **Валідація:** Вводьте реалістичні дані. Якщо це email — обов'язково з `@`, якщо LinkedIn — валідне посилання `https://linkedin.com/in/...`, якщо це зарплата ("Від" і "До") — переконайтеся, що `from < to`.

### Текстові поля (Textarea)
Часто в `textarea` ставлять специфічні питання. Аналізуйте `placeholder` (приводьте до `downcase`) і відповідайте відповідно:
- Якщо є "date" -> `ASAP`.
- Якщо є "Yes or No" -> `Yes`.

### Завантаження файлів (CV)
Сайт може валідувати розмір або формат файлу прямо на фронтенді (наприклад, "Minimum file size 1Kb").
- **Рішення:** Завжди генеруйте тестовий файл достатнього розміру (наприклад, > 1024 байт), щоб обійти фронтенд-перевірки.
- **Дія:** `input[type="file"].set_input_files('dummy_cv.pdf')`.

### Складні випадаючі списки (Vuetify Custom Selects) - 🔴 Найскладніший етап
Стандартний `page.select_option` **не працює** з Vuetify (бо це не `<select>`, а набір `div`). Використання JavaScript (клік по прихованим елементам списку) часто дає збій, оскільки меню рендериться в кінці `<body>` або залишається невидимим.

**Найкращий підхід (Координатний клік):**
1. Знайдіть шапку селектора (напр., `.select-head`).
2. Клікніть по ній, щоб відкрити меню. Додайте `sleep 1`, щоб дочекатися закінчення анімації Vuetify.
3. Розрахуйте координати меню відносно шапки і клікніть мишкою (`page.mouse.click`).
```ruby
head = dropdown.locator('.select-head')
head.click
sleep 1 # Чекаємо відкриття меню
box = head.bounding_box
# Клікаємо нижче від інпута (+70px), пропускаючи можливий варіант "Not set"
page.mouse.click(box['x'] + 20, box['y'] + box['height'] + 70)
```

## 3. Сабміт та зчитування результату

### Сабміт форми
Кнопка відправки може не мати `type="submit"`. Шукайте за текстом (Send, Відправити, Submit) або беріть останню кнопку в формі.

### Перехоплення алертів (Vue Toasts / Alerts)
Після сабміту Vue-сайти не перезавантажують сторінку. Вони можуть показати тост (`.v-snack`), алерт (`.v-alert`) або просто змінити текст на сторінці.

**Як правильно чекати:**
Уникайте очікування за *системними класами* (наприклад, `alerts.first.wait_for(state: 'visible')`), оскільки у DOM може знаходитися прихований шаблон алерту, і Playwright нескінченно чекатиме його появи, ігноруючи реальні зміни на сторінці.

**Правильний підхід:** Шукати **безпосередньо очікуваний текст** про успіх (exact text) і динамічно чекати його (з хорошим таймаутом, оскільки запит може бути довгим).
```ruby
puts "Очікуємо до 30 секунд..."
success_msg = page.locator('text="Your application has been sent"')
success_msg.wait_for(state: 'visible', timeout: 30000)
```

## 4. Обхід перевірок Cloudflare та Google reCAPTCHA v3

Найпідступніший момент — коли форма ідеально заповнена, але після натискання `Send` кнопка починає "крутитися" і зависає назавжди. Це означає, що **reCAPTCHA v3 у фоновому режимі (без малюнків) розпізнала бота і заблокувала XHR запит**, а фронтенд не знає, як це обробити.

**Щоб успішно отримати токен та пройти перевірку, потрібен комплексний Stealth-підхід:**

1. **Persistent Context (Постійний профіль):**
   Використовуйте `launch_persistent_context`. Браузер з історією та кукісами отримує набагато вищий "Human Score" від Google, ніж пуста анонімна сесія.
2. **Реальний браузер (Channel: Chrome):**
   Використовуйте `channel: 'chrome'`, а не стандартний Chromium від Playwright, щоб мати реальні відбитки браузера.
3. **Вимкнення Automation Flags:**
   Передавайте `args: ['--disable-blink-features=AutomationControlled']`, щоб прибрати позначки, за якими Cloudflare ідентифікує ботів.
4. **Скрипти підміни (Init Scripts):**
   До того, як сторінка завантажиться (`add_init_script`), необхідно підмінити маркери автоматизації:
   - Видалити `navigator.webdriver`.
   - Зімітувати плагіни (`navigator.plugins`: Chrome PDF Viewer тощо).
   - Підмінити WebGL Vendor/Renderer (наприклад, `Google Inc. (NVIDIA)` -> реальна відеокарта).
   - Зімітувати `window.chrome`.
5. **Людська поведінка (Емуляція пауз):**
   Між кліками та введенням тексту додавайте випадкові затримки (`sleep rand(0.1..0.4)`). Занадто швидке заповнення складної форми — це "червоний прапорець" для reCAPTCHA.

**Висновки:** Комбінація *реального Chrome, збереженого профілю, stealth-підмін та координатної взаємодії з UI* дозволяє стабільно автоматизувати навіть найбільш захищені SPA-додатки.
